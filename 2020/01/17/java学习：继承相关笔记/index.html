重新阅读”extend”的小点

  1. super：书本概念：是一个指示编译器调用超类方法的特殊关键词。（个人理解:是调用父类相关的关键词调用父类构造器，方法和成员变量）。

  2. 构造器：书本上允许隐式构造父类构造器，但是实践发现隐式调用的前提是父类必须显式的写出默认空构造器(虽然自己写空构造器本身是比较好的习惯)。

  3. 方法调用：

    1. 方法的名字和参数是标签，而返回类型则不算，所以要保证返回类型的兼容性。
    2. 动态绑定依赖于隐式参数的实际类型，为了减少调用时检索的开销，所以会为每个类创建方法表。
    3. private final static方法使用的是静态绑定(还需多斟酌)。

  4. 类型转换：当需要将父类对象通过强制类型转换成子类对象的时候，如果忽略父类对象的具体引用，编译会抛出java.lang.ClassCastException的错误(例：employ对象如果指向的是Manager对象，则可以强制转换，如果指向的是本身Employee类型的则会出错)。可以在每次转换前加instance of的条件判断，不过最正确的做法应该是少用类型转换，更多是将类设计的更合理，通过动态绑定等方法来实现功能。
