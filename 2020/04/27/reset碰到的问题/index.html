git reset碰到的问题

Apr 27, 2020 | git

写代码的时候想多多使用git版本管理工具来管理代码，今天突然想好好的学习一下版本回退功能，结果就git reset这条语句就想了一个小时才算入门。

git reset版本回退语句，功能是将工作区，暂存区（index），以及head指向的版本库进行版本回退。基本用法：git reset 版本号 --hard（or mixed）(指定路径）
备注：版本号不填默认上一次提交版本，指定路径不填默认当前工作区，--不填默认mixed

  1. git reset （版本号）--soft，将版本库的head指针指向指定版本，暂存区和工作区不变
  2. git reset （版本号）--mixed，将版本库的head指针指向指定版本，并且将上个版本文件同步到暂存区（说白就是清除暂存区）。
  3. git reset （版本号）--hard，此操作最危险，将head指针指向指定版本，并且将指定版本文件同步到暂存区和工作区。

注意：以上操作都会移动head指针，也就是会取消若干个提交记录，当某个文件在之前的版本未出现的话，回退之后会显示此文件未被追踪，也就是没有被记录进工作区，曾困扰许久。
