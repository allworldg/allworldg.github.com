我的第一篇博客文章（创建过程）

Nov 30, 2019

概述

如果想创建个人博客，又无从下手的情况下，Hexo是个好选择。Hexo是一个基于Node.js的用于制作简洁且高效博客的博客框架。Hexo使用Markdown标记语言来编写文本，然后利用配置好的主题样式生成静态网页。

  • hexo是制作博客的一个工具
  • 它将我们用markdown语法写的文章生成html页面，然后通过搭载自己的服务器上传到网上。

第一步安装

要想写blog，首先我们需要准备好以下两个工具

  • Node.js（运行在服务器的JavaScript）
  • npm（包管理工具，解决Nodejs的部署问题）
  • Git（一个开源的分布式管理系统，里面的git bash以后也可以代替cmd来使用）

安装完后，git的配置在此先不谈，如果有需要的请通过Google或者度娘解决，因为之后需要借助GitHub服务器来进行博客的上传，所以请先配置好git。

安装Node.js的时候，可以任意选存放地址，这次我安装选择的是D盘。安装完成后，可以在自己系统的命令行窗口输入

npm -v

node -v

来查看当前npm和Nodejs的版本号，如果没有正常显示说明安装有问题或者环境变量设置的问题。检查完之后，因为npm包是国外的版本，在之后上传搭载可能比较耗时，所以我们需要下载国内淘宝的镜像文件，在命令行内输入下面一句代码

npm install -g cnpm --registry=https://registry.npm.taobao.org

进行下载，下载好后输入cnpm -v来查看版本号。安装完cnpm镜像文件后，我们就可以通过cnpm来下载hexo相关包了，在命令行内输入cnpm install -g hexo-cli进行全局下载（全局下载默认是c盘，想修改默认位置可以参考这个博客https://www.cnblogs.com/liaojie970/p/9296177.html ，进阶版就是在局部安装hexo包，详情参考hexo官方文档)，下载完输入命令hexo -v来查看hexo版本号。在顺利完成以上步骤，我们基础的配置要求就已经设置完成了，如果有出现问题实在解决不了的可以删除重装，反正软件都不大。

第二步 创建博客并且发布到本地

  1. 设置一个目录作为以后写博客的位置。首先在自己指定的位置建立一个文件夹（我的文件夹名设置为blog），然后在当前目录下用命令行（命令行位置一定要指向当前文件夹）输入命令 hexo init来初始化hexo的配置，（需要网络，可能会有点卡，可以耐心等待，如果中途有卡住不动，可以敲击一下键盘或者点一下鼠标 试试。）下载成功后会发现文件夹内多了很多自动生成的文件，在此先不仔细分析，感兴趣的话可以自己摸索。

  2. 初始化好之后可以在本地运行一下hexo的服务，输入命令hexo s或者hexo server，生成本地的服务，在网页上输入hexo生成的指定端口号之后，可以看到生成的博客网页。然后在命令行下输入 hexo n "文章名字",就可以在 blog\source_posts 目录下面找到刚刚生成的md文件，用文本编辑器打开，使用markdown语法就可以进行博客的书写了。写完保存，然后在blog目录下命令行输入hexo g 来进行文章的html文件生成并且发布到主页上。构建成功后使用hexo s来查看一下自己刚写的文章是否在博客上显示。

  3. 提到的常用语句 hexo init hexo n "new title name" hexo g hexo s

    以上就是第二大步所需要的完整步骤

第三步 从本地发布到GitHub服务器上

如果我们想将博客发布到网上，又不想去考虑流量安全问题，可以选择现有的免费的服务器进行搭载，在这里我使用的是GitHub服务器，接下来介绍流程。
